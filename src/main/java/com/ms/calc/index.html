Troubleshooting

A list of issues you may run into and how to resolve them:
  1. Plugins are not installing and showing errors
    Restart Jenkins from Desktop folder user_repo > Utility. Check the plugins in the installed tab of manage jenkins. If plugins are not available in it, then install the same from the available tab

  2. Jenkins job is throwing a "JAVA_HOME varaible not set" error
    Follow the steps in Configuration 2 to set the java and maven variables in Jenkins

  3. Jenkins job is throwing a "pom.xml not found" error
    Make sure that the following sequence of task execution is done in every job (except the first):
    1. Copy artifacts from previous job
    2. Execute the maven commands

  4. Deploy option is not available
    Make sure the Deploy plugin was uploaded using the Advanced tab. Same goes for any other option that may not be displayed in the job configurations

  5. Copying artifacts from previous job is throwing an error
    Any of the following options can be tried:
    1. Ensure that the previous job exists and is not spelt incorrectly
    2. Try deleting the existing job and creating it once again
    3. Restart Jenkins

  6. SmokeTest job is failing: Deployment failed to tomcat
    Any of the following options can be tried:
    1. Make sure that the latest tomcat version was choosen from the dropdown inside the Deploy war/ear to container task.
    2. Ensure that the credentials used were exactly as given on the table present on the Context | Configure tools tab.
    3. Check if Tomcat service is running by checking the URL mentioned on the same table.

  7. Eclipse is showing errors under Markers tab before executing maven goals
    Ignore those errors and proceed with executing the maven goals as given in the Exercises Tab
   

DevOps Using Open Source Tool Stack

Context: There is a Calculator application provided to you in Eclipse. It is a web application written using jsp. There is a pom.xml provided which contains the various Maven build lifecycle targets defined. Unit test cases are also provided to you. index.jsp is the landing page of the web application.

Note: If you would like to go through learning materials and videos related to this course, please refer to the Lex course for Infosys DevOps Java Certification here

Objective: Create a Jenkins pipeline with the following jobs:

  • CI initialization – invoking clean, compile targets
  • CI quality – invoking sonar, Junit, code coverage targets
  • CI publish – invoking war,publishing to artifactory and smoke test
Steps:
  1. Use the project as a local Git repository and commit changes on the local branch.
  2. Configure the build using Maven for invoking sonar, JUnit, jacoco, creation of war file and baselining war version to artifactory.
  3. Run the Maven build in Eclipse and observe the results
  4. Configure jobs in Jenkins for the various targets provided earlier
  5. Create a pipeline view of the jobs
  6. Schedule a 2 minute build (Poll SCM option)
  7. Trigger a build and observe the pipeline view
  8. Go back to Eclipse and make some UI changes (or any changes to code) and commit the code back to the local git repository. The build pipeline in Jenkins should get triggered automatically
  9. Open SonarQube (http://localhost:9000). Add custom profile and custom gating conditions as learnt in the demos. Give values so that you can observe a broken and a smooth build
  10. Trigger a build in Jenkins and observe the gating condition in SonarQube working.
  11. Create a powershell script to check the unit test code coverage and fail if expected coverage is not achieved. Add this to code coverage job
  12. Make changes to the unit tests (so that some of them fail) and observe the gating conditions and build failing in Jenkins. Correct them and observe that the build becomes stable. (Make changes and commit the test code from Eclipse)
  13. Trigger a build in Jenkins and observe the gating condition script working for code coverage job.

    14. Summary of this Exercise: You have learnt to use Jenkins for continuous integration!
